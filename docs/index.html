长篇小说 / 章节叙事 / 创意写作

摘要

长文本生成面临结构漂移、角色不一致与主题偏移等问题。我们构建了一个协同式 RAG-Agent 系统，将剧情、人物、世界观、检索、评估与写作能力模块化，并通过 动态记忆与一致性检测形成闭环，使模型在多轮生成中维持可解释的创作轨迹。 系统支持章节化与整篇模式，可通过参数控制长度、节奏与评估频率，并在实际输出中 提供可追溯的过程记录与可复现的配置入口。

关键词
多 Agent RAG 动态记忆 一致性评估 长文本生成

主要贡献

协同式多 Agent 结构

将规划、写作、检索、评估解耦为独立 Agent，支持可插拔与分阶段执行。

动态记忆与一致性闭环

按章节写入向量记忆，并通过一致性检查与实时修订回写改进内容。

可复现的生成流程

记录大纲、轮次摘要、评估得分与输出文本，便于对比实验与追溯。

方法概览

01

任务分析

解析主题、类型与目标长度，建立可控的生成约束。

02

全局与章节大纲

生成全局叙事结构，并分解为章节级执行计划。

03

多 Agent 生成

写作 Agent 生成文本，检索与世界观 Agent 提供支撑材料。

04

一致性评估与修订

评估生成内容，自动修订与回写记忆，形成闭环。

系统架构

从任务分析、全局大纲、章节生成到一致性评估与修订的完整闭环。

评估与日志

自动评估

从连贯性、情感强度与整体质量等维度进行评估，并记录分数。

过程可追溯

按轮次存储摘要、转折点与一致性问题，便于复盘与对比实验。

实验资料

复现与配置

1. 安装依赖

pip install -r requirements.txt

2. 配置 API Key

set DEEPSEEK_API_KEY=your_key
# 或者
set OPENAI_API_KEY=your_key
set CODEX_API_KEY=your_key

3. 运行生成

python main.py
提示：可在 .env 中设置环境变量；更多参数见下方配置表。

限制与伦理

内容一致性

长篇叙事仍可能出现风格漂移与局部逻辑跳跃，需要人工审校。

计算成本

多 Agent 与检索流程对算力与 API 配额有较高需求。

内容安全

建议结合平台内容安全策略与人工审核使用生成文本。

GitHub Pages 部署

  1. 将本项目推送到 GitHub 仓库。
  2. 进入仓库设置 → Pages → 选择 docs/ 作为发布目录。
  3. 访问 https://xiao-zi-chen.github.io/Long-Story-agent/ 即可查看页面。
  4. 如需自定义域名，创建 docs/CNAME 并写入域名。
