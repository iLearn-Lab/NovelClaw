Overview

TL;DR

Problem

Long-form text often drifts in plot logic, character behavior, and pacing after multiple iterations.

Approach

We decompose writing into specialist agents and connect them with retrieval, dynamic memory, and consistency checking.

Outcome

The workflow improves controllability, traceability, and repeatability for long narrative generation.

System

Workflow

The pipeline starts from task analysis and outline planning, then alternates generation, retrieval, evaluation, and revision. Updated summaries are written back to memory for the next iteration.

Method

Agent Workflow

Stage 1

Plan

Analyzer and Organizer define constraints, goals, and chapter-level plans.

Stage 2

Generate

Writer collaborates with Plot, Character, and World agents for chapter drafting.

Stage 3

Retrieve + Remember

Retriever and static knowledge modules inject context; memory stores key updates.

Stage 4

Evaluate + Revise

Evaluator and consistency checker trigger targeted revision before the next loop.

Main Contributions

  • Modular multi-agent architecture for long-form writing.
  • Closed-loop memory and consistency control.
  • Practical reproducibility with detailed run artifacts.

Observed Strengths

  • Improved chapter-level continuity through memory write-back.
  • Automatic correction reduces obvious contradiction drift.
  • Run logs allow direct debugging and comparison across settings.

Current Limits

  • Generation length may still undershoot target words.
  • Performance depends on model quality and API budget.
  • Human review is still recommended for final publication.

Reproducibility

Quick Start

Install

pip install -r requirements.txt

Set API Keys

set DEEPSEEK_API_KEY=your_key
# or
set OPENAI_API_KEY=your_key
set CODEX_API_KEY=your_key

Run

python main.py

Outputs are stored in runs/; configuration files are in config.py and config.toml.
